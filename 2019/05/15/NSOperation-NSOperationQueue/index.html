NSOperation & NSOperationQueue

发表于 2019-05-15 | 分类于 iOS

简介

NSOperation、NSOperationQueue 是一套OC的多线程解决方案，实际上NSOperation是对GCD的封装，面向对象并且提供一些高级的api，可以实现任务之间的依赖，最大并发任务数，取消任务等高级功能。参考2015年的 WWDC Advanced NSOperations。

用法

  • NSOperation可以单独使用，创建后直接调用start方法，系统同步执行操作。
  • 配合NSOperationQueue 实现异步执行，实现步骤：
    1. 创建NSBlockOperation，或者是NSOperation的子类
    2. 创建 NSOperationQueue 对象
    3. NSOperation 对象添加到 NSOperationQueue 对象中
      系统就会自动将 NSOperationQueue 中的 NSOperation 取出来，在新线程中执行操作。

特性

  1. NSOperation可以设置依赖

    NSBlockOperation *operation2 = [NSBlockOperation blockOperationWithBlock:^{
    NSLog(@"excute operation2");        
    }];
NSBlockOperation *operation1 = [NSBlockOperation blockOperationWithBlock:^{
     NSLog(@"excute operation1");
  }];
[operation2 addDependency:operation1];
[queue addOperation:operation2];
[queue addOperation:operation1];

  2. 可以执行cancel操作

    - (void)cancel;
- (void)cancelAllOperations;

  3. 可以指定优先级

    @property NSOperationQueuePriority queuePriority;

  4. NSOperationQueue设置最大并发数

    @property NSInteger maxConcurrentOperationCount;

  5. operation执行完成时回调

    @property (nullable, copy) void (^completionBlock)(void) API_AVAILABLE(macos(10.6), ios(4.0), watchos(2.0), tvos(9.0));

查看Swift的Operation、OperationQueue的源码实现。

swift中的Operation和OperationQueue就对应OC里面的NSOperation和NSOperationQueue，可以通过查看swift corelibs foundation的源码来学习operation的功能实现。

Operation.swift

  1. 首先来看看operation的基本属性：

    operation属性里包含着几部分内容：

    • 互斥锁
    • weak形式的OperationQueue
    • 一堆状态变量
    • 依赖集合_dependencies
    • 一堆DispatchGroup
    • 优先级相关的链表“指针”

  1. start/main方法

    NOTE: 基类的main方法是一个空方法，start方法中调用main方法，修改_executing的值，最后执行finish方法

  2. cancel方法

    NOTE: cancel只是设置一个标记，更新_cancelled的值

  3. add/remove dependency

    NOTE:
    addDependency 把依赖的op，加入到自己的 _dependencies 中，自己的 _depGroup执行enter操作 ，并且把_depGroup加入到op的 _groups 中
    removeDependency 把依赖的op，从到自己的 _dependencies 中移除，然后把op的_groups中自己的 _depGroup移除并且执行leave操作

  4. finish方法

    NOTE: finish是个内部方法，对_groups的所有的_depGroup执行leave操作，告诉自己加入的 queue 已经结束了
    执行completionBlock，使用global的queue执行。

到这里operation的基本的源码就说完了，从简单的源码中可以得出的结论

  • operation的依赖功能，就是对dispatchgroup功能的封装，通过group的enter，leave，wait（queue里面用到）基本的能力来实现的。

  • 简单的operation子类可以重写main方法实现基本的功能

  • 需要自己控制operation的生命周期的，可以重写start方法，并且不要执行父类的方法。
  • cancel操作只能cancel掉未执行start方法的operation，如果想cancel掉正在执行中的operation，需要自己在main方法中判断

OperationQueue调度

  1. operationQueue的基本属性

operation queue属性里包含着几部分内容：

* 互斥锁
* 信号量Semaphore
* __underlying DispatchQueue
* queueGroup DispatchGroup 
* DispatchWorkItem数组
* _OperationList（存放着各种优先级的operation）

  1. addOperation方法

这段addOperations的方法是queue中关键的方法，下面进行解读：

  • waitGroup是内部创建的dispatch group，主要作用是阻塞当前线程，实现waitUntilFinished功能
  • ops是准备加入queue的operation，对每个op设置他们的queue为self，并且把他们都加到_operations中，这里的加入过程涉及到优先级问题
  • 对ops中的每个operation，执行返回值是 DispatchWorkItem 的闭包，items中包含的是DispatchWorkItem的数组
  • 在_underlyingQueue中执行每一个DispatchWorkItem，workItem的主要工作：
    • 如果waitGroup存在，对waitGroup执行enter操作，这个也比较好理解，如果需要waitUntilFinished，就要对每个op进行enter，在结束的时候在进行leave操作。
    • _concurrencyGate 信号量实现operation queue的并发控制，控制执行_runOperation的数量
    • 执行_runOperation方法

  1. _runOperation方法

    • _operations执行dequeue操作，返回一个合适的op
    • 在执行_waitUntilReady和start方法前都加入了isCancelled判断

    • 执行_waitUntilReady，结合上面operation中对 depGroup 的解析，可以知道这是在等待依赖的operation结束，一旦依赖结束，或者是没有依赖关系，就会马上进入 ready 状态。

    • 执行op的start方法

OperationQueue的调度功能基本上就说完了，不管是operation，还是operationQueue中的waitUntilFinished功能都是 DispatchGroup 来实现的，queue的并发功能是通过 信号量 来实现。operation的优先级功能也是queue通过_OperationList自定义的结构体进行管理调度的。

疑问

  • 为什么对isCancelled,isFinished,isReady的读操作都不加锁，而isExecuting的读操作是加锁的

  • BlockOperation中对main方法重写，如果直接执行他的start方法，executionBlocks不像是在子线程执行的
